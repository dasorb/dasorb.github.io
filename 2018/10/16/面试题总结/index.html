面试题总结

发表于 2018-10-16 | 分类于 面试

基础篇

解释一下面向对象的特征

面向对象的特征有封装、继承、多态

封装：是将对象关键特征和属性，封装到对象内部，与外界隔离。保证程序的健壮性

继承：子类继承父类具有的特征，有利于代码的复用性

多态：不同类对象对相同行为的不同反应

static语句块与非static语句块、构造器之间是怎样加载的

有父类的情况：

先加载父类的static语句块->加载子类的static语句块-> 父类非static语句块->父类构造器

->子类非static语句块->子类构造器

方法的重写和方法的重载的区别和条件

重写：是子类对父类方法的一种重写，只能比父类抛出的异常更少，访问权限不能比父类小。不能重写private修饰的方法

重载：是同一个类中，有两个或两个以上相同方法名，但参数列表各不相同的方法，它们之间构成重载。其中参数列表中：参数的个数，参数的顺序，参数的类型至少有一个不同。

继承有什么缺点吗

子类继承父类必须考虑父类具体的实现细节，重写方法，破坏了java语言的封装性。

解释面向对象和面向过程语言的特点

面向过程语言，分析的是解决问题的具体步骤。

面向对象，是将问题中的各个事务分解成对象，然后描述对象在问题中的具体行为。

==和equals的区别

基本类型之间使用==来判断是否相等。

引用类型，用==是判断两者之间是否引用的是同一个对象。用equals是判断两者之间的等价性，如果一个类没有重写equals，默认使用的是父类object的equals方法。

hashcode和equals的区别和联系

equals和hashcode方法都是判断两个对象是否相等一致，但是重写的equals里比较全面，比较复杂，所以效率比较低，而利用hashCode方法进行对相比，则只要生成一个hash值就可以了，效率很高。但是hashcode并不是完全可靠，有时候不同的对象他们生成的hashCode也会一样。而equals对比则是很可靠的，所以他们俩可以配合使用。equals相等，哈希值一定相等，哈希值相等，equals不一定相等。

描述一下jvm加载class文件的机制

jvm负责类的加载，连接，初始化。java中类的加载是由类加载器和他的子类完成，类的加载器会把.class文件的数据存入到一个字节数组中，然后产生与之相对应的.class对象爱过你，之后jvm进入连接阶段，会进行验证，准备，和解析操作，然后jvm对其进行初始化。

string不可变的好处

可以用作hashmap的key。

因为不可变，所以在使用并发的时候，使用string是线程安全的。

new String (“xyz“)创建了几个字符串对象

创建了两个字符串对象，一个是在静态区，一个是new在堆中的对象，其中堆中的对象指向静态区的字符串

java中支持的基本数据类型有哪些

java中有8大基本数据类型：整型有byte short int long 浮点型有float double 字符型char 布尔型boolean

其中byte占1字节，short占2字节，int占4字节，long占8字节，float占4字节，double占8字节，char占两个字节，布尔占1字节

什么自动装箱和自动拆箱

java中基本类型都有与之对的包装类，目的是方便类型之间的相互转换，装箱就是将基本数据类型转换为包装类，拆箱就是把包装类转换为基本类型

Integer缓存池的问题

java中整型自动装箱时，在-128-127范围之间转换时都是调用的缓存池中的对象

抽象类和接口的区别

抽象类可以定义普通的成员变量，而接口定义的是static final修饰的变量

抽象类只能单继承和finnal可以多继承。

抽象类主要表现的是”是不是“的关系，而接口表现的是”有没有“概念。

使用的时候，一般用抽象类定义成员变量，实现接口来扩展功能。

集合篇

常见的集合有哪些

常见的集合有两个大接口collection和map

collection有两个常见的子接口：list和set一个表示有序集，一个无序集

list常见的实现类有：快查询ArrayList，快插入LinkedList，vector

set常见的实现类有：hashSet，treeSet

map常见的实现类有：hashmap、treemap，hashTable，properties，concurrentHashMap

ConcurrentHashMap和HashTable和HashMap的区别

hashMap线程不安全，但是比hashTable存取效率高。hashTable线程安全。concurrentHashMap结合两者的优势，将hash默认分为16个桶，每次进行操作，之锁住当前用到的桶

hashMap的长度为什么是2的幂次方

因为哈希表中，key的定位是用元素的hash值 “与” 哈希表的长度-1,如果与的是偶数那么2进制最后一位永远是0，这样位置最后一位为1的空间使用不到。所以使用2的幂次减小浪费，减少重读，提高查询的效率。

ArrayList、HashSet、HashMap的初始容量

ArrayList初始容量为10，当元素超过一半进行扩容，扩容大小是0.5倍+1

HashSet和HashMap的初始容量为16，当元素超过0.75扩容，扩容大小是原容量的1倍

List和Set区别

list的实现类有序且元素可重复

set的实现类无序且元素不可重复，无序指插入顺序，和取出顺序不一致。

多线程篇

线程和进程的概念、并行和并发的概念

当一个程序调入内存中运行后，就成了进程，进程指进行中的程序。

一个进程中，可能包含多个顺序执行流，每个顺序执行流就是一个线程。

并发，同一时刻只能有同一条指令运行，但由于CPU轮换执行速度，在宏观上看就有多条指令同时执行。

并行，同一时刻多条指令在多个处理机上同时执行。

进程间通信的方式

传统的进程间通信的方式：管道，命名管道，信号量，消息队列，共享内存，socket

java中可以采用的方法：socket，rmi,webservice,webServler。

线程间的通信可以直接传入共享的变量来实现。

创建线程的三种方式

继承Thread父类创建线程，实现Runnable接口创建线程，实现Callable接口

线程安全

当多个线程访问某个方法时，不管通过怎样的调用方式，如何交替执行，我们在主程序中不需要去做任何的同步，这个类的结果行为和我们设想的结果行为相同，那么这个方法就是线程安全的。

线程的生命周期

新建(New)、就绪(Runnable)、运行(Running)、阻塞(Blocked)和死亡(Dead)5种状态

新建：当创建Thread类的要给实例时，此线程进入新建状态。

就绪：线程已经启动，正在等待被分配给cpu时间片。

运行：线程获得CPU资源正在执行run方法。

阻塞：由于某种原因导致正在运行的线程让出CPU并暂停自己的执行，即进入堵塞状态。

死亡：当线程执行完毕或被其他线程杀死，线程就进入死亡状态，这是线程不可能再进入就绪状态等待执行。

javaweb篇

session和cookie的区别

cookie数据存放在客户端的浏览器上，session数据存放于服务器上，所以cookie不太安全，而session比较安全。

session会随着会话的增多，占用更多的服务器性能，而cookie不会。

cookie的保存的数据不能超过4k。

Http请求中的post和get方式的区别

post方式比get方式更安全，因为get方式会直接作为url的一部分进行提交。而post不会被缓存。

get方式做为url的一部分，所以会有url长度的限制，post的数据更大。

但是post比get的请求方式慢，所以一般用post用于数据的写入和修改。get用于排序和筛选

MVC思想

mvc是一种软件架构的思想，将一个软件按照模型、视图、控制器来进行划分。

模型用来封装业务逻辑，视图用来实现表示逻辑，控制器用来协调模型与视图。

模型处理的业务逻辑要交给控制器，控制器选择视图来进行显示，视图要通过控制器来调用模型。

框架篇

spring的IOC和AOP

ioc 控制反转，将对象的依赖关系交给spring容器，使用配置文件来创建所以依赖的对象，由主动创建对象改为被动获取对象。

aop：面向切面编程，将功能代码从业务逻辑中分离出来。

aop的实现方式有哪几种？如何选择

有两种，分别是jdk动态代理实现和cglib实现，如果目标对象实现了接口，默认采用jdk动态代理实现，可以强制使用cglib。如果目标对象没有实现接口，必须采用cglib库进行实现。

jdk动态代理如何实现？

创建一个代理类实现InvocationHadler接口，使用Proxy.newProxyInstance()方法生成代理对象

springMVC的核心控制器是什么？消息处理流程有哪些

核心处理器为DispatchServlet。

首先请求来到前端控制器，会根据映射处理器HandlerMapping，调用相应的Controller

Controller返回一个ModelAndView数据加视图名对象。根据视图解析器调用页面，然后显示在浏览器中。

springMVC运行流程

  1. 用户发送请求到DispatchServlet

  2. DispatchServlet根据请求路径查询具体的Handler

  3. HandlerMapping返回一个HandlerExcutionChain给DispatchServlet

    HandlerExcutionChain：Handler和Interceptor集合

  4. DispatchServlet调用HandlerAdapter适配器

  5. HandlerAdaper调用具体的Handler处理业务

  6. HandlerAdaper处理结束返回一个具体的ModelAndView给适配器

    ModelAndView：model->数据模型，view->视图名称

  7. 适配器将ModelAndView给DispatchServlet。

  8. DispathServlet把视图名称交给视图解析器ViewResolver

  9. ViewResolver返回一个具体的视图给DispatchServlet

  10. 渲染视图，展示给用户

mybatis中#和$占位符的区别？

使用#当占位符，可以提前预编译语句，相当于使用jdbc中的preparedStatement，可以防止sql注入问题，将传入的参数当字符串处理。而$不会当作字符串处理，是什么就是什么。
