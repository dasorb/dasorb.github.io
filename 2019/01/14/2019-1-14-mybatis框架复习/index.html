mybatis框架复习

发表于 2019-01-14 | 分类于 框架学习

mybatis介绍

Mybatis是一个基于持久层的框架，提供自动映射，动态SQL，级联，缓存，注解，代码和SQL分离等特性

mybatis有四个核心组件：

  1. SqlSessionFactoryBuilder(通过构建者模式，创建SqlSessionFactory)
  2. SqlSessionFactory(一般为单例，负责创建SqlSession)
  3. SqlSession(读取接口)
  4. SqlMapper(映射器)

SqlSessionFactory有两个具体实现类，DefaultSqlSessionFactory,SqlSessionManager。单线程时默认使用前者，多线程时使用后者。

在mybatis中，xml分为两类：

  1. 是基础配置文件，负责配置最基本上下文参数和运行环境。
  2. 是映射文件，它可以配置映射关系，SQL，参数等信息。

组件的生命周期：

  1. SqlSessionFactoryBuilder负责创建SqlSessionFactory创建后生命周期结束。
  2. SqlSessionFactory可看作一个数据库连接池，创建SqlSession接口对象，可看作mybatis的生命周期。
  3. SqlSession可看作数据库连接对象，存活在一个业务请求当中，请求结束归还SqlSessionFactory。
  4. Mapper代表请求中的一个业务处理，随着SqlSession的关闭而废弃它。

mybatis简单使用

xml创建流程:

  1. 在基本配置文件中，<configuration>标签中通过<typeAlias>可以为相关类配置别名，简化使用。
  2. 通过<environments>中<transactionManager>配置事务管理，<dataSource>配置数据库连接。
  3. 通过<mappers>标签可以导入相关mapper映射xml文件。

mybatis全局配置文件

  1. <properties>，可以根据resource指向的配置文件，用${}引入外部属性配置。
  2. <typeAliases>，在此标签下可以配置别名。用简单的名称，代替全类名
  3. <envoronments>，配置数据库信息
    • 在此标签下可以配置<transactionManager>事务管理器，JDBC或MANAGED
    • 配置数据源相关<dataSource>，有POOLED,UNPOOLED,JNDI。也可以实现DataSourceFactory接口引入自定义数据源。
  4. <databaseProvider>数据库厂商标识，在多数据库切换时，可以使用别名进行区分。
    • 使用方式：<databaseProvider type="DB_VENDOR">设置别名，在映射器的标签中dataBaseId使用使用。
  5. <mappers>标签导入mapperXML文件。

mybatis映射器使用

select元素

  1. <select>元素负责查询SQL语句的编写
    • id：是唯一标识
    • paramterType：表示这条SQL接受的参数类型，别名/全限定名
    • resultType：表示这条SQL返回的结果类型，别名/全限定名
  2. 多个参数进行传递：三种方式：Map，@param注解，JavaBean。
    • 参数少推荐使用注解，参数多使用JavaBean。
  3. 使用resultMap映射结果集
    • id定义了唯一标志，type代表使用哪个类去映射。
  4. 分页参数RowBounds
    • 支持分页，只需要给接口添加RowBounds参数即可，创建类时指定偏移量，显示条数。

insert/updata/delete元素

  1. 在执行一条insert语句之后，会返回一个整数表示其影响记录数。
  2. 主键回填：拿到插入数据后的主键
    • useGeneratedKeys控制是否获取主键，keyProperty/keyColumn指定生成的主键。
  3. 自定义主键，自定义主键增加规则，<selectKey>标签
    • keyProperty采用哪个属性作为主键，resultType返回结果集，order指定将于定义的SQL之前或之后运行

SQL元素

抽取常用的列名：

  1. 使用<sql>标签，使用id指定唯一标识。
  2. 在使用时，<include refid = "">，引用标识。

resultMap元素

定义sql到javaBean之间的关系。

id配置主键，result配置其他列。

属性property定义projo属性，column定义sql字段，两者对应。

可以使用<constructor>配置POJO对象。

级联

级联：例如角色需要存在用户与之对应，一个角色可能有多个用户，这就是一对多级联。

级联的分类：

  1. 鉴别器：<discriminator>，类似与switch case的用法
  2. 一对一：<association>
  3. 一对多：<collection>
  4. property代表映射的pojo属性；column代表SQL的列，用作传递给select属性的参数；select是命名空间+SQL id的形式。

N+1问题

N+1：假设现在有N个关联完成了级联，那么只要在加入一个关联关系，就编程了N+1级联，所有的级联SQL都会被执行，会造成很大的资源浪费，这就是N+1问题。

为了应对这类问题，提供了延迟加载功能

  • 在setting中，lazyLoadingEnabled延迟加载的全局开关，aggeressiveLazyLoading开启层级加载，关闭则是按需加载。
  • fetchType属性，使用它可覆盖全局配置，在mapper中的元素里使用。值eager代表获得当前pojo立即加载对应的数据，lazy获得当前pojo后延迟加载对应的数据。

缓存

mybatis有一级缓存和二级缓存之分，一级缓存属于session级别，二级缓存属于SqlSessionFactory级别。一级缓存默认使用，开启二级缓存的方法：在mapper文件中使用<cache/>即可。

自定义缓存：

  • mybatis支持自定义缓存，要求缓存类实现接口org.apache.ibatis.cache.Cache
  • 在<cache>标签中，使用type指定缓存类的全类名。
  • 还可以自定义是否进行缓存，属性：flushCache代表是否刷新缓存，useCache是select特有的，代表是否需要使用缓存。
  • 对于其他mapper文件使用自定义缓存，还可以使用<cache-ref>来指定。

动态SQL

IF元素

一般常与test属性搭配使用，根据条件是否构建包含的SQL语句。

choose、when、otherwise元素

这三者之间的功能类似与：swich…case…default…

trim、where、set元素

where可以把语句中多余的and去除，进行条件的拼接

trim要去除包含SQL中的一些特殊的字符串，prefix代表语句的前缀，prefixOverrides去除多余的前缀。suffixOverrides去除多余的后缀。

set可以在update元素代替set关键字，同时也会去除多余的逗号。

foreach元素

用于遍历一个传过来的集合参数。一般不用

bind元素

通过OGNL表达式去自定义一个上下文的变量，自定义字符串拼接，属性name定义标识，value定义值。

<bind name="pattern" value="'%' + _parameter + '%'"
